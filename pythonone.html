Q1. Who developed Python? (Asked in 2022)

Theory: Guido van Rossum began implementing Python in December 1989 as a successor to the ABC language. He wanted a language that was easy to read, simple to use, and powerful enough for real-world tasks. He is often referred to as the "Benevolent Dictator for Life (BDFL)" of Python, a title he held until he stepped back from the role in 2018.

Real-Life Example: Just as Dennis Ritchie developed C and James Gosling developed Java, Guido van Rossum single-handedly designed Python's philosophy and syntax.

Important Points:

  • Creator: Guido van Rossum
  • Nationality: Dutch
  • Organization: CWI, Netherlands
  • Title given: BDFL (Benevolent Dictator for Life)

PYQ Connection: Directly asked in 2022 objective section; may repeat as 1-mark or fill-in-the-blank question.

Practice Questions:

2. In which institute was Python first developed?

Quick Revision: Guido van Rossum = Creator of Python (BDFL).

Q3. Python is a High-Level Programming Language. Explain why this statement is correct. (Asked in 2024)

Technical Definition: A high-level language is one that is written in a human-readable form (close to English) and is independent of machine/hardware architecture, requiring an interpreter or compiler to convert it into machine code.

Theory — Reasons Python is High-Level:

  1. Simple, English-like syntax — no need for complex symbols like ; or {} in most cases.
  2. Abstraction from hardware — programmer doesn't manage memory manually (automatic garbage collection).
  3. Portability — same code runs on Windows, Linux, Mac without change.
  4. Interpreted execution — Python interpreter converts code line-by-line into machine code.
  5. Rich built-in libraries reduce the need to write low-level logic.

Real-Life Example: Writing in Python is like giving instructions in plain English to an assistant (interpreter) who then figures out how to physically do the task (machine-level execution) — you don't need to explain every micro-detail.

Python Program: # High-level, simple syntax example name = "Rahul" print("Hello,", name) Output: Hello, Rahul

Important Points: High-level = closer to human language | Low-level = closer to machine (Assembly, Machine code) | Python hides hardware-level complexity | Needs an interpreter (CPython) to run.

Frequently Asked
Exam Point of View: Give definition + minimum 3 reasons with example for full marks.
Quick Revision: Python is high-level → simple syntax + portability + no hardware handling + interpreted.

Q4. An Expression is a combination of values, variables and operators. Explain. (Asked in 2024)

Technical Definition: An expression in Python is any legal combination of values (constants), variables, and operators that evaluates to produce a single value.

Theory - Types of expressions:

  1. Arithmetic Expression → a + b * 2
  2. Relational Expression → a > b (returns True/False)
  3. Logical Expression → a > 5 and b < 10
  4. String Expression → "Hello" + "World"

Every expression, when evaluated by Python, returns a value that can be stored in a variable or printed.

Real-Life Example: Just like a mathematical formula Area = length × width combines variables and an operator to give a result, a Python expression works the same way.

Python Program: a = 10 b = 5 expr1 = a + b # arithmetic expression expr2 = a > b # relational expression expr3 = (a > 5) and (b < 10) # logical expression print(expr1, expr2, expr3) Output: 15 True True

Important Points: Expression always returns a value | Made up of: values + variables + operators | Different from a "statement," which performs an action but may not return a value.

Shortcut
Quick Revision: Expression = combination that evaluates to a value; types = arithmetic/relational/logical/string.

Q5. What are Mutable Data Types in Python? (Asked in 2024)

Technical Definition: Mutable data types are those whose value/content can be changed after creation, without changing their memory address (identity).

Theory:

  • Mutable types: list, dictionary, set
  • Immutable types (opposite): int, float, string, tuple, bool

When we modify a mutable object, its id() (memory address) stays the same. When we try to "modify" an immutable object, Python actually creates a new object in memory.

Real-Life Example: A mutable object is like a whiteboard — you can erase and rewrite on it while it remains the same board. An immutable object is like a printed page — to change the text, you must print a new page.

Python Program: # Mutable example - list my_list = [1, 2, 3] print("Before:", my_list, id(my_list)) my_list.append(4) print("After:", my_list, id(my_list)) # id remains same # Immutable example - string my_str = "hello" print("Before:", my_str, id(my_str)) my_str = my_str + " world" print("After:", my_str, id(my_str)) # id changes Output: Before: [1, 2, 3] 140234567891200 After: [1, 2, 3, 4] 140234567891200 Before: hello 140234567892345 After: hello world 140234567893456

(Actual id numbers will vary by system.)

Important Points: Mutable: list, dict, set | Immutable: int, float, string, tuple, bool, frozenset | id() function checks memory address.

Most Important
Exam Point of View: Always mention id() function and give one program-based proof.
Quick Revision: Mutable = list/dict/set (changeable); Immutable = int/float/str/tuple (unchangeable).

Q7. Which of the following is an incorrect variable declaration? Explain the rules of variable declaration. (Asked in 2023)

Technical Definition: A variable is a named location in memory used to store data values. Python has specific rules that must be followed to create a valid (legal) variable name.

Theory — Rules for Variable Naming in Python:

  1. Must start with a letter (a-z, A-Z) or underscore (_) — cannot start with a digit.
  2. Can contain letters, digits, and underscores only (no spaces or special characters like @, #, %, -).
  3. Cannot be a Python keyword (e.g., if, for, class).
  4. Case-sensitive: Name and name are different variables.
  5. No spaces allowed (use underscore instead, e.g., first_name).
Example of Incorrect Declarations: 1name = "Ram" # ❌ Invalid — starts with digit first name = 10 # ❌ Invalid — contains space class = "A" # ❌ Invalid — 'class' is a keyword first-name = "Sita" # ❌ Invalid — hyphen not allowed Example of Correct Declarations: name1 = "Ram" # ✅ Valid _first_name = "Sita" # ✅ Valid firstName = "Geeta" # ✅ Valid

Real-Life Example: Just as a person's name in official documents cannot start with a number or contain symbols, Python variable names must follow strict identifier rules.

Memory Trick
Rule: Letter/underscore start → only letters/digits/underscore → not a keyword → case-sensitive.

Q8. Explain Membership Operators in Python with suitable examples. (Asked in 2024 - Subjective Trend)

Technical Definition: Membership operators are used to test whether a value or variable is found in a sequence (string, list, tuple, set, dictionary). Python provides two membership operators: in and not in.

OperatorDescriptionExample

Real-Life Example: Checking if a student's roll number exists in the attendance list — in operator works exactly like that search.

Python Program: fruits = ["apple", "banana", "mango"] print("banana" in fruits) print("grapes" in fruits) print("grapes" not in fruits) # with string text = "Python Programming" print("Prog" in text) Output: True False True True
Shortcut

Q9. Solve the following Operator-based MCQs and explain the correct answers. (Asked in 2022)

Technical Definition: This question tests understanding of Python operators — arithmetic, relational, logical, assignment, bitwise, and identity/membership operators — through multiple-choice evaluation.

Theory — Operator Categories:

  • Arithmetic: + - * / // % **
  • Relational: == != > < >= <=
  • Logical: and or not
  • Assignment: = += -= *= /=
  • Identity: is, is not
  • Membership: in, not in

Sample MCQ Practice with Explanation:

Q. What is the output of 17 // 3?
a) 5.67 b) 5 c) 6 d) 2
Answer: b) 5 — // is floor division; it discards the decimal part.

Q. What is the output of 17 % 3?
a) 5 b) 2 c) 5.6 d) 0
Answer: b) 2 — % gives the remainder (17 = 5×3 + 2).

Q. What is the output of 2 ** 3?
a) 6 b) 8 c) 9 d) 5
Answer: b) 8 — ** is exponentiation (2³ = 8).

Memory Trick
Operator Precedence: () > ** > * / // % > + - > relational > logical.
Master floor division (//), modulus (%), and exponent (**) — most repeated in MCQs.

Q11. What is the correct file extension of a Python program? Explain its importance. (Asked in 2024)

Technical Definition: A Python source code file is saved with the extension .py. Other related extensions include .pyc (compiled bytecode), .pyw (Windows GUI scripts), and .ipynb (Jupyter notebooks).

Importance of .py extension:

  1. Identifies the file as Python source code for OS and IDEs.
  2. Enables the Python interpreter to recognize and execute the file directly.
  3. Allows IDEs (like PyCharm, VS Code) to provide syntax highlighting and error detection.
  4. Needed for import statements — modules are imported using the file name without .py.

Real-Life Example: Just like .docx tells the OS to open a file in Word, .py tells the OS/interpreter to treat the file as Python code.

Shortcut
Quick Revision: .py = source code, .pyc = compiled bytecode, .pyw = GUI script (no console). Simply write .py with one-line reasoning — quick, easy marks.

Q2. Describe the installation process of Python and explain Python Path. (Asked in 2022)

Technical Definition: Installation refers to setting up the Python interpreter on a computer system so that Python programs can be written and executed. The Python Path (PATH environment variable) tells the operating system where to find the Python executable file so it can be run from any location in the command line/terminal.

Theory — Steps to Install Python (Windows):

  1. Visit the official website: python.org/downloads
  2. Download the latest stable installer for your OS.
  3. Run the installer (.exe file).
  4. Important: Check the box "Add Python to PATH" before clicking Install.
  5. Click "Install Now" and wait for setup to complete.
  6. Verify installation by opening Command Prompt and typing: python --version

Explaining Python Path:

  • PATH is a system environment variable listing directories where executable programs are located.
  • If Python's installation folder is added to PATH, typing python in any terminal window directly launches the interpreter.

Q6. Explain Lambda Function with suitable examples. (Asked in 2024)

Syntax: lambda arguments: expression

SECTION C: EXPECTED QUESTIONS

Q1. Explain the Features of Python with suitable examples.

Theory — Major Features:

  1. Simple and Easy to Learn — English-like syntax, minimal punctuation.
  2. Free and Open Source — freely downloadable; source code can be modified.
  3. Interpreted Language — executes code line-by-line; no separate compilation step.
  4. Portable — same code runs on Windows, Linux, macOS without modification.
  5. Object-Oriented — supports classes, objects, inheritance.
  6. High-Level Language — abstracts hardware-level details.
  7. Extensive Standard Library — built-in modules.
  8. Dynamically Typed — no need to declare variable types explicitly.
  9. Extensible — can integrate with C, C++, Java code.

Real-Life Example: Python's versatility is like a Swiss Army knife — one tool usable for web development, automation, data science, and AI.

Q2. Explain Variables, Identifiers and Keywords in Python.

  • Variable: Named memory location to store data. Created automatically when value is assigned (e.g., x = 10).
  • Identifier: Name used to identify a variable, function, class, etc. Must start with letter/underscore, case-sensitive.
  • Keyword: Reserved word with predefined meaning (e.g., if, else, for). Python has 35 keywords.

Q3. Explain Type Conversion in Python with suitable examples.

Technical Definition: Type conversion is the process of converting a value from one data type to another.

1. Implicit Type Conversion:
Performed automatically by Python during expressions to avoid data loss (e.g., int → float).

2. Explicit Type Conversion (Type Casting):
Performed manually using built-in functions: int(), float(), str(), etc.

Python Program: # Implicit conversion a = 10 # int b = 5.5 # float c = a + b # int automatically converted to float print(c, type(c)) # 15.5 <class 'float'> # Explicit conversion x = "25" y = int(x) + 10 # string converted to int explicitly print(y, type(y)) # 35 <class 'int'>
